# Fix My Bad Website

Use this when a page should feel like the actual product, not generic AI output.

Start by understanding the product and the page. Do not jump straight to gradients, cards, giant heroes, or decorative filler.

## Rules

- Inspect the actual files and rendered page when possible.
- Identify the page's job before changing visuals.
- Make the first screen useful; do not build a fake marketing shell when the user needs the actual tool.
- Use real product signals: screenshots, data, controls, states, examples, assets, or working UI.
- Avoid samey AI defaults: vague hero copy, floating card piles, purple-blue glow, oversized whitespace, and decorative blobs.
- Keep controls ergonomic and readable on mobile and desktop.
- Pair with `plain-web-copy` when text is part of the problem.
- Verify with a browser or screenshot when the app needs rendering proof.

## Output

Report:

- `Problem:` what made it generic or confusing.
- `Change:` what changed and why.
- `Proof:` screenshot, browser check, or command.
- `Remaining:` anything still unverified visually.
